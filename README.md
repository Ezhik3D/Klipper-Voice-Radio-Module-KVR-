# Klipper Voice & Radio Module (KVR)

Интеллектуальный аудио-модуль для Klipper, превращающий ваш 3D-принтер в полноценный медиа-центр. Поддерживает голосовые уведомления с системой очередей, интернет-радио с динамическими метаданными и автоматический режим тишины.

---

## 🚀 Основные возможности

*   **Система очередей (Cascade)**: Уведомления проигрываются последовательно. Радио автоматически отключается на время речи и возвращается по завершении всей очереди.
*   **Динамические метаданные**: Вывод названия радиостанции, жанра и текущего трека на KlipperScreen и в консоль (через систему "Шпион" с помощью `ffprobe`).
*   **Автономность и Smart Reconnect**: Интеллектуальный мониторинг интернет-соединения. При обрыве связи радио отключается, а при восстановлении — автоматически возобновляет вещание.
*   **Memory State**: Модуль запоминает последнюю громкость, активный поток в файл `voice_state.json`. После перезагрузки всё восстановится автоматически.
*   **Quiet Hour (Тихий час)**: Автоматическое отключение звука в заданное время, очистка очередей уведомлений и автоматическое пробуждение системы утром.
*   **Smart Volume**: Голосовые уведомления всегда звучат на 100% мощности канала, радио — на заданном вами уровне.

## 🛠 Установка (Manual Installation)

### 1. Установка зависимостей
```bash
sudo apt update
sudo apt install -y mpg123 alsa-utils ffmpeg pulseaudio pulseaudio-module-bluetooth bluez
```

### 2. Настройка прав доступа
```bash
sudo usermod -aG audio,pulse-access,bluetooth klipper
```
*Обязательно перезагрузите систему после этого шага.*

### 3. Подготовка папок и файлов
```bash
mkdir -p ~/voice_files
cp voice.py ~/klipper/klippy/extras/
```

### 4. Настройка конфигурации
Добавьте секцию `[voice]` в ваш `printer.cfg`.
Настройка зависит от выбранного типа вывода звука.

```ini
[voice]
path: ~/voice_files/         # Путь к папке с вашими mp3 файлами
default_volume: 80           # Громкость при первом запуске модуля (0-100)
work_start: 08:00            # Время начала работы звука (HH:MM)
work_end: 22:00              # Время ухода в режим тишины (HH:MM)
stations:                    # Список URL радиостанций через запятую
    http://ep256.hostingradio.ru:8052/europaplus256.mp3,
    http://rusradio.hostingradio.ru/rusradio128.mp3,
    http://nashe1.hostingradio.ru/nashespb128.mp3

# --- ВАРИАНТ А: ПРОВОДНЫЕ ДИНАМИКИ (ALSA) ---
output_type: alsa            # Прямой вывод через звуковую карту
control_card: 0              # Индекс карты (проверить через 'aplay -l')
control_device: Master       # Имя канала (проверить через 'amixer scontrols')

# --- ВАРИАНТ Б: BLUETOOTH ИЛИ ПРОВОД ЧЕРЕЗ PULSEAUDIO ---
# output_type: pulse         # Вывод через звуковой сервер PulseAudio
# control_device: @DEFAULT_SINK@ # Авто-выбор текущего активного устройства
# ВАЖНО: для BT можно указать конкретное имя, например: bluez_sink.12_11_06_D8_7A_17.a2dp_sink
```

#### Подробное описание параметров:

*   **output_type**: 
    *   `alsa`: Используется в чистых системах без графики или там, где не установлен PulseAudio. Минимальная задержка, прямой доступ к железу.
    *   `pulse`: Требуется для работы Bluetooth-колонок. Позволяет нескольким программам использовать звук одновременно.
*   **control_card** (только для ALSA): Индекс физической аудиокарты. Обычно `0` — встроенный звук Orange Pi, `1` — USB-карта.
*   **control_device**: 
    *   В режиме `alsa` это обычно `Master`, `Speaker` или `Headphone`.
    *   В режиме `pulse` используйте `@DEFAULT_SINK@`, чтобы модуль управлял громкостью того устройства, которое сейчас выбрано в системе по умолчанию.
*   **stations**: Вы можете добавить неограниченное количество станций. Если их больше 300, модуль при первом запуске начнет "лениво" собирать их названия в фоновом режиме (по 1 станции раз в 3 секунды), чтобы не нагружать процессор принтера.

Для вывода сообщений на KlipperScreen у вас должна быть включена секция:
```ini
[respond]
```

## 🎧 ПОДРОБНАЯ ИНСТРУКЦИЯ ПО ПОДКЛЮЧЕНИЮ BLUETOOTH КОЛОНКИ (OPI3 LTS)

Чтобы колонка работала автоматически как системная служба, сама подключалась при включении и была доступна для Klipper через Unix-сокет, выполните следующие шаги строго по порядку:

### ШАГ 1: Права доступа
Разрешаем пользователю klipper управлять звуком и Bluetooth:
```bash
sudo usermod -aG pulse-access,audio,bluetooth klipper
```

### ШАГ 2: Настройка аудио-моста (PulseAudio)
Нужно прописать модули в системный конфиг, чтобы они создавали сокет для Klipper при каждом запуске:
1. Откройте файл: `sudo nano /etc/pulse/default.pa`
2. Добавьте в самый конец файла эти 3 строки:
```text
load-module module-bluetooth-policy
load-module module-bluetooth-discover
load-module module-native-protocol-unix auth-anonymous=1 socket=/tmp/pulse-socket
```
*(Нажми Ctrl+O, Enter для сохранения и Ctrl+X для выхода)*

### ШАГ 3: Настройка авто-подключения (System)
Чтобы Bluetooth-стек сам "бегал" за колонкой:
1. Откройте файл: `sudo nano /etc/bluetooth/main.conf`
2. Найдите и раскомментируйте (уберите `#`) строки:
   * `AutoEnable=true`
   * `ReconnectAttempts=0` (в самом конце файла)
   * `FastConnectable=true`
3. **Перезагрузите систему**: `sudo reboot`

### ШАГ 4: Сопряжение (bluetoothctl)
Выполняется один раз вручную:
```bash
bluetoothctl
power on
agent on
default-agent
scan on
# Дождись появления адреса колонки (например 12:11:06:D8:7A:17)
pair 12:11:06:D8:7A:17
trust 12:11:06:D8:7A:17
connect 12:11:06:D8:7A:17
exit
```

### ШАГ 5: Установка устройства по умолчанию
Указываем системе приоритет вывода звука:
1. Узнаем имя устройства: `pactl list sinks short`
2. Сделайте колонку дефолтной (замените имя на своё):
```bash
pactl set-default-sink bluez_sink.12_11_06_D8_7A_17.a2dp_sink
```

### ШАГ 6: Проверка
Убедитесь, что "дверь" для Klipper открыта:
```bash
ls -l /tmp/pulse-socket
```
Если файл подсвечен как сокет — всё готово. В `printer.cfg` достаточно указать `output_type: pulse`.

### ШАГ 7: Настройка авто-подключения при старте (для Orange Pi)
Если колонка не подключается автоматически после перезагрузки, добавьте команду принудительного соединения в автозагрузку системы:

1. Узнайте MAC-адрес вашей колонки (если забыли):
   ```bash
   bluetoothctl devices
   ```
   *Адрес выглядит как `12:11:06:D8:7A:17`.*

2. Откройте файл автозагрузки:
   ```bash
   sudo nano /etc/rc.local
   ```

3. Добавьте следующую строку перед `exit 0` (заменив MAC-адрес на свой):
   ```bash
   (sleep 15 && bluetoothctl connect 12:11:06:D8:7A:17) &
   ```

4. Сделайте файл исполняемым:
   ```bash
   sudo chmod +x /etc/rc.local
   ```

## 🕹 Команды G-Code


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `VOICE S=имя` | Воспроизвести `имя.mp3` из папки звуков. | `VOICE S=finished` |
| `FM S=номер` | Включить радио из списка по номеру. | `FM S=1` |
| `FM URL=ссылка`| Включить радио по прямой ссылке. | `FM URL=http://...` |
| `FM_STOP` | Выключить радио (стирает URL из памяти). | `FM_STOP` |
| `FM_LIST` | Показать список всех станций в консоли. | `FM_LIST` |
| `SET_VOLUME S=X`| Установить громкость от 0 до 100. | `SET_VOLUME S=50` |
| `CLEAR_VOICE` | Очистить очередь и прервать текущий голос. | `CLEAR_VOICE` |

## 💡 Полезные советы (Макросы)

1. **Макросы**: Пример макросов для удобного управления короткими командами:

```ini
#####################################################################
# ГОРЯЧИЕ КЛАВИШИ И УПРАВЛЕНИЕ ЗВУКОМ
#####################################################################

[gcode_macro FM]
variable_last_s: 1
rename_existing: FM_BASE
gcode:
  {% if params.S is undefined %}
    # Циклическое переключение станций
    {% set S = printer["gcode_macro FM"].last_s + 1 %}
    {% if S > 3 %} # Укажите своё количество онлайн радиостанций
      {% set S = 1 %}
    {% endif %}
  {% else %}
    {% set S = params.S|int %}
  {% endif %}
  
  SET_GCODE_VARIABLE MACRO=FM VARIABLE=last_s VALUE={S}
  FM_BASE S={S}

[gcode_macro FM_STOP]
rename_existing: FM_STOP_BASE
gcode:
  FM_STOP_BASE

[gcode_macro FM_NEXT]
gcode:
  FM

[gcode_macro FM_PREVIOUS]
gcode:
  {% set S = printer["gcode_macro FM"].last_s - 1 %}
  {% if S < 1 %}
    {% set S = 3 %} # Укажите своё количество онлайн радиостанций
  {% endif %}
  FM S={S}

#####################################################################
# УПРАВЛЕНИЕ ГРОМКОСТЬЮ
#####################################################################

[gcode_macro SET_VOLUME]
rename_existing: SET_VOLUME_BASE
gcode:
  {% if params.S is undefined %}
    # Если нажать просто SET_VOLUME, громкость прибавляется на 5% по кругу
    {% set current_vol = printer["voice"].default_volume|default(80)|int %}
    {% set S = current_vol + 5 %}
    {% if S > 100 %}
      {% set S = 0 %}
    {% endif %}
  {% else %}
    {% set S = params.S|int %}
  {% endif %}
  SET_VOLUME_BASE S={S}

[gcode_macro VOLUME_UP]
gcode:
  {% set current_vol = printer["voice"].default_volume|default(80)|int %}
  {% set S = current_vol + 5 %}
  {% if S > 100 %}{% set S = 100 %}{% endif %}
  SET_VOLUME S={S}

[gcode_macro VOLUME_DOWN]
gcode:
  {% set current_vol = printer["voice"].default_volume|default(80)|int %}
  {% set S = current_vol - 5 %}
  {% if S < 0 %}{% set S = 0 %}{% endif %}
  SET_VOLUME S={S}

#####################################################################
# СЕРВИСНЫЕ КОМАНДЫ (Глушилки)
#####################################################################

[gcode_macro VOICE_OFF]
gcode:
    CLEAR_VOICE
    FM_STOP
    M118 Sound system disabled (all muted)

[gcode_macro CLEAR_VOICE]
rename_existing: CLEAR_VOICE_BASE
gcode:
    CLEAR_VOICE_BASE
```

2. **Автоматизация**: Используйте `VOICE S=start_print` в вашем макросе `START_PRINT`, чтобы принтер голосом сообщил о начале работы.
3. **Тихий старт**: Если принтер включается в "тихий час", громкость автоматически будет 0%, а радио не запустится до утра.

# Документация модуля Klipper Voice & Radio (KVR)

## 1. ПРИНЦИП РАБОТЫ ГОЛОСОВЫХ УВЕДОМЛЕНИЙ (VOICE)
Голосовая система построена на базе плеера `mpg123` и использует строгую систему очередей.
*   **Очередность**: Не перемешивает уведомления, а играет их строго друг за другом (каскад).
*   **Приоритет Громкости**: На время речи модуль принудительно ставит громкость на 100%.
*   **Обнуление**: `CLEAR_VOICE` мгновенно очищает очередь и убивает текущий процесс речи.

## 2. ПРИНЦИП РАБОТЫ РАДИО (FM) И МЕТАДАННЫХ

Радиосистема использует `ffplay` для воспроизведения и `ffprobe` для получения данных о треках.

### Взаимодействие с Голосом:
*   **Авто-пауза**: Если играет радио и поступает команда `VOICE`, радио моментально выключается (процесс убивается), чтобы освободить аудиокарту.
*   **Авто-возврат**: Как только очередь голоса становится пустой, модуль ждет от 0.5 до 1.2 секунды (пауза для ALSA) и автоматически запускает последнюю игравшую радиостанцию.
*   **Вызов FM во время каскада**: Если вы переключаете станцию (`FM S=...`), пока принтер говорит, модуль обновляет URL в памяти. Музыка включится сама, как только принтер замолчит.

### Контроль соединения (Smart Reconnect):
*   **Мониторинг**: Система постоянно проверяет наличие интернета. При обрыве радио отключается, чтобы не занимать ресурсы, и выводит уведомление о потере сети.
*   **Автоподключение**: Как только интернет восстанавливается, модуль автоматически перезапускает поток, возвращая музыку без участия пользователя.

### Метаданные и KlipperScreen:
*   **Live-Парсинг**: При подключении модуль извлекает `icy-name` (станция) и `icy-genre` (жанр).
*   **Обновление треков**: Поток-шпион раз в 15-30 секунд вызывает `ffprobe`, забирает актуальный `StreamTitle` и выводит его: `🎵 Сейчас играет: ...`.
*   **Вывод на экран**: Все сообщения дублируются на KlipperScreen через систему `RESPOND`. Чтобы интерфейс не игнорировал пачки уведомлений, внедрена система адаптивных задержек (1.0с для станции, 2.0с для жанра и 3.0с для названия трека). Это гарантирует, что каждое сообщение будет отображено.

## 3. ПАМЯТЬ СОСТОЯНИЙ (MEMORY STATE)

Модуль обладает энергонезависимой памятью, реализованной через файл `~/voice_state.json`.

*   **Что запоминается**: Последний установленный уровень громкости и URL последней активной радиостанции.
*   **Запись**: Данные обновляются мгновенно при командах `SET_VOLUME`, `FM` или `FM_STOP`.
*   **Восстановление**: При старте Klipper (или после `FIRMWARE_RESTART`) модуль читает JSON. Если перед выключением радио играло — оно включится само.
*   **FM_STOP**: Если вы выключили радио вручную командой `FM_STOP`, оно **не включится** само при перезагрузке, так как URL в памяти будет стерт.

---

## 4. РЕЖИМ ТИШИНЫ (QUIET HOUR)

Модуль имеет встроенный планировщик, работающий по системным часам.

*   **Настройка**: Параметры `work_start` и `work_end` определяют окно, когда звук разрешен.
*   **Наступление ночи**: Как только время переходит границу `work_end`, модуль:
    1. Мгновенно убивает радио и голос.
    2. Полностью очищает очередь уведомлений (чтобы они не "всплыли" утром).
    3. Сбрасывает системную громкость в 0.
*   **Наступление утра**: В момент `work_start` модуль проверяет, было ли включено радио перед уходом в тишину. Если да — музыка начинает играть автоматически.
*   **Игнорирование команд**: Любые команды `VOICE`, пришедшие ночью, будут проигнорированы. Команда `FM` ночью лишь сохранит станцию в память, но не запустит звук.
*   Формат времени должен состоять из двух цифр часа и двух цифр минут, например 06:05

## 5. ПРИМЕРЫ СЦЕНАРИЕВ


| Ситуация | Поведение системы |
| :--- | :--- |
| **Играет FM, вызван VOICE** | FM тухнет -> Голос говорит на 100% -> FM возвращается сам. |
| **Вызван каскад из 10 VOICE** | Все 10 проигрываются по очереди -> Включается FM. |
| **Пропал интернет во время FM** | Уведомление о потере сети -> Процесс убивается до появления связи. |
| **Интернет восстановился** | Система обнаруживает сеть -> Автозапуск последней станции. |
| **Старт радио (Метаданные)** | Поэтапный вывод на экран (Станция -> Жанр -> Трек) с задержками. |
| **Смена трека (Шпион)** | `ffprobe` находит новое название -> Уведомление на ClipperScreen. |
| **VOICE говорит, вызван FM_STOP** | URL стирается из памяти. После конца речи FM не включится. |
| **VOICE говорит, вызван CLEAR_VOICE** | Речь обрывается. Если FM был активен — он вернется через 1 сек. |
| **Наступило время Work_End** | Радио обрывается, очередь VOICE чистится. Полный Mute (0%). |
| **Включение принтера днем** | Авто-восстановление громкости и станции из JSON-памяти. |
| **Включение принтера ночью** | Загрузка в режиме тишины (0%), автозапуск FM заблокирован до утра. |

*   Плееры: `mpg123`, `ffmpeg` (ffplay, ffprobe).
*   Доступ: Пользователь Klipper должен состоять в группах `audio` и `pulse-access`.
*   Конфиг: Обязательно наличие секции `[voice]` и `[respond]`.

################################################################################
# ПРОВЕРКА РАБОТОСПОСОБНОСТИ
################################################################################
1. Проверь наличие сокета: `ls -l /tmp/pulse-socket`
2. Проверь громкость в консоли Klipper: `SET_VOLUME S=100`
3. Запусти тестовый голос: `VOICE S=имя_файла`

**ЧТО ДЕЛАТЬ, ЕСЛИ ЗВУКА НЕТ:**
1. Проверь в bluetoothctl статус: `Connected: yes`
2. Тест в терминале: `mpg123 -o pulse ~/voice_files/восемь.mp3`
3. Убедись, что создан сокет `/tmp/pulse-socket`.
