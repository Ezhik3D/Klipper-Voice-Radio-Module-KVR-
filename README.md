# Klipper Voice & Radio Module (KVR)

Интеллектуальный аудио-модуль для Klipper, превращающий ваш 3D-принтер в полноценный медиа-центр. Поддерживает голосовые уведомления с системой очередей, интернет-радио с динамическими метаданными и автоматический режим тишины.

---

## 🚀 Основные возможности

*   **Система очередей (Cascade)**: Уведомления проигрываются последовательно. Радио автоматически отключается на время речи и возвращается по завершении всей очереди.
*   **Динамические метаданные**: Вывод названия радиостанции, жанра и текущего трека на ClipperScreen и в консоль (обновление через систему "Шпион" с помощью `ffprobe`).
*   **Автономность и Smart Reconnect**: Интеллектуальный мониторинг интернет-соединения. При обрыве связи модуль корректно завершает процесс, сообщает об ошибке, а при восстановлении интернета — автоматически возобновляет вещание.
*   **Memory State**: Модуль запоминает последнюю громкость и активный поток в файл `voice_state.json`. После перезагрузки всё восстановится автоматически.
*   **Quiet Hour (Тихий час)**: Автоматическое отключение звука в заданное время, очистка очередей уведомлений и автоматическое пробуждение системы утром.
*   **Smart Volume**: Голосовые уведомления всегда звучат на 100% мощности канала, радио — на заданном вами уровне.

## 🛠 Установка (Manual Installation)

Выполните следующие шаги в терминале вашего хоста:

### 1. Установка зависимостей
```bash
sudo apt update
sudo apt install -y mpg123 alsa-utils ffmpeg
```

### 2. Настройка прав доступа
```bash
sudo usermod -aG audio \$USER
```
*Рекомендуется перезагрузить систему после этого шага.*

### 3. Подготовка папок и файлов
```bash
mkdir -p ~/voice_files
cp voice.py ~/klipper/klippy/extras/
```

### 4. Настройка конфигурации
Добавьте в ваш `printer.cfg`:

```ini
[voice]
path: ~/voice_files/         # Путь к папке с mp3
default_volume: 80           # Громкость при первом запуске (0-100)
control_card: 0              # Номер звуковой карты (aplay -l)
control_device: Master       # Канал управления (amixer scontrols)
work_start: 08:00            # Начало работы (HH:MM)
work_end: 22:00              # Конец работы (HH:MM)
stations:                    # Список URL через запятую
    http://ep256.hostingradio.ru:8052/europaplus256.mp3,
    http://rusradio.hostingradio.ru/rusradio128.mp3,
    http://nashe1.hostingradio.ru/nashespb128.mp3
```
Для вывода сообщений на KlipperScreen у вас должно быть включено`:

```ini
[respond]
```

## 🕹 Команды G-Code


| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `VOICE S=имя` | Воспроизвести `имя.mp3` из папки звуков. | `VOICE S=finished` |
| `FM S=номер` | Включить радио из списка по номеру. | `FM S=1` |
| `FM URL=ссылка`| Включить радио по прямой ссылке. | `FM URL=http://...` |
| `FM_STOP` | Выключить радио (стирает URL из памяти). | `FM_STOP` |
| `FM_LIST` | Показать список всех станций в консоли. | `FM_LIST` |
| `SET_VOLUME S=X`| Установить громкость от 0 до 100. | `SET_VOLUME S=50` |
| `CLEAR_VOICE` | Очистить очередь и прервать текущий голос. | `CLEAR_VOICE` |

## 💡 Полезные советы (Макросы)

Пример макросов для удобного управления короткими командами:

```ini
#####################################################################
# ГОРЯЧИЕ КЛАВИШИ И УПРАВЛЕНИЕ ЗВУКОМ
#####################################################################

[gcode_macro VOICE]
variable_last_id: 1
rename_existing: VOICE_BASE
gcode:
  {% if params.S is undefined %}
    # Циклическое переключение файлов, если S не указан
    {% set S = printer["gcode_macro VOICE"].last_id + 1 %}
    {% if S > 10 %} # Укажите своё количество файлов .mp3
      {% set S = 1 %}
    {% endif %}
  {% else %}
    {% set S = params.S %}
  {% endif %}
  
  SET_GCODE_VARIABLE MACRO=VOICE VARIABLE=last_id VALUE={S}
  VOICE_BASE S={S}

[gcode_macro FM]
variable_last_s: 1
rename_existing: FM_BASE
gcode:
  {% if params.S is undefined %}
    # Циклическое переключение станций
    {% set S = printer["gcode_macro FM"].last_s + 1 %}
    {% if S > 3 %} # Укажите своё количество онлайн радиостанций
      {% set S = 1 %}
    {% endif %}
  {% else %}
    {% set S = params.S|int %}
  {% endif %}
  
  SET_GCODE_VARIABLE MACRO=FM VARIABLE=last_s VALUE={S}
  FM_BASE S={S}

[gcode_macro FM_STOP]
rename_existing: FM_STOP_BASE
gcode:
  FM_STOP_BASE

[gcode_macro FM_NEXT]
gcode:
  FM

[gcode_macro FM_PREVIOUS]
gcode:
  {% set S = printer["gcode_macro FM"].last_s - 1 %}
  {% if S < 1 %}
    {% set S = 3 %} # Укажите своё количество онлайн радиостанций
  {% endif %}
  FM S={S}

#####################################################################
# УПРАВЛЕНИЕ ГРОМКОСТЬЮ
#####################################################################

[gcode_macro SET_VOLUME]
rename_existing: SET_VOLUME_BASE
gcode:
  {% if params.S is undefined %}
    # Если нажать просто SET_VOLUME, громкость прибавляется на 5% по кругу
    {% set current_vol = printer["voice"].default_volume|default(80)|int %}
    {% set S = current_vol + 5 %}
    {% if S > 100 %}
      {% set S = 0 %}
    {% endif %}
  {% else %}
    {% set S = params.S|int %}
  {% endif %}
  SET_VOLUME_BASE S={S}

[gcode_macro VOLUME_UP]
gcode:
  {% set current_vol = printer["voice"].default_volume|default(80)|int %}
  {% set S = current_vol + 5 %}
  {% if S > 100 %}{% set S = 100 %}{% endif %}
  SET_VOLUME S={S}

[gcode_macro VOLUME_DOWN]
gcode:
  {% set current_vol = printer["voice"].default_volume|default(80)|int %}
  {% set S = current_vol - 5 %}
  {% if S < 0 %}{% set S = 0 %}{% endif %}
  SET_VOLUME S={S}

#####################################################################
# СЕРВИСНЫЕ КОМАНДЫ (Глушилки)
#####################################################################

[gcode_macro VOICE_OFF]
gcode:
    CLEAR_VOICE
    FM_STOP
    M118 Sound system disabled (all muted)

[gcode_macro CLEAR_VOICE]
rename_existing: CLEAR_VOICE_BASE
gcode:
    CLEAR_VOICE_BASE
```

2.  **Автоматизация**: Используйте например `VOICE S=start_print` в вашем макросе `START_PRINT` где start_print имя вашего файла .mp3, чтобы принтер голосом сообщил о начале работы.
3.  **Тихий старт**: Если принтер включается в "тихий час", громкость автоматически будет установлена в 0%, а радио не запустится до наступления рабочего времени.

# Документация модуля Klipper Voice & Radio (KVR)

Этот модуль представляет собой интеллектуальную аудиосистему для управления голосовыми уведомлениями и интернет-радио напрямую из интерфейса Klipper.

---

## 1. ПРИНЦИП РАБОТЫ ГОЛОСОВЫХ УВЕДОМЛЕНИЙ (VOICE)

Голосовая система построена на базе плеера `mpg123` и использует строгую систему очередей.

### Поведение очереди (Каскад):
*   **Очередность**: Если Klipper отправляет несколько команд `VOICE` подряд, модуль не перемешивает их. Он создает очередь и проигрывает файлы один за другим.
*   **Приоритет Громкости**: В момент начала речи модуль принудительно выставляет системную громкость на 100% (канал `control_device`), чтобы уведомление было четко слышно. По окончании всей очереди громкость возвращается к пользовательскому значению.
*   **Обнуление очереди**: Команда `CLEAR_VOICE` мгновенно очищает список ожидающих файлов и убивает текущий процесс воспроизведения.

---

## 2. ПРИНЦИП РАБОТЫ РАДИО (FM) И МЕТАДАННЫХ

Радиосистема использует `ffplay` для воспроизведения и `ffprobe` для получения данных о треках.

### Взаимодействие с Голосом:
*   **Авто-пауза**: Если играет радио и поступает команда `VOICE`, радио моментально выключается (процесс убивается), чтобы освободить аудиокарту.
*   **Авто-возврат**: Как только очередь голоса становится пустой, модуль ждет от 0.5 до 1.2 секунды (пауза для ALSA) и автоматически запускает последнюю игравшую радиостанцию.
*   **Вызов FM во время каскада**: Если вы переключаете станцию (`FM S=...`), пока принтер говорит, модуль обновляет URL в памяти. Музыка включится сама, как только принтер замолчит.

### Контроль соединения (Smart Reconnect):
*   **Мониторинг**: Система постоянно проверяет наличие интернета. При обрыве радио отключается, чтобы не занимать ресурсы, и выводит уведомление о потере сети.
*   **Автоподключение**: Как только интернет восстанавливается, модуль автоматически перезапускает поток, возвращая музыку без участия пользователя.

### Метаданные и ClipperScreen:
*   **Live-Парсинг**: При подключении модуль извлекает `icy-name` (станция) и `icy-genre` (жанр).
*   **Обновление треков**: Поток-шпион раз в 15-30 секунд вызывает `ffprobe`, забирает актуальный `StreamTitle` и выводит его: `🎵 Сейчас играет: ...`.
*   **Вывод на экран**: Все сообщения дублируются на ClipperScreen через систему `RESPOND`. Чтобы интерфейс не игнорировал пачки уведомлений, внедрена система адаптивных задержек (1.0с для станции, 2.0с для жанра и 3.0с для названия трека). Это гарантирует, что каждое сообщение будет отображено.

---

## 3. ПАМЯТЬ СОСТОЯНИЙ (MEMORY STATE)

Модуль обладает энергонезависимой памятью, реализованной через файл `~/voice_state.json`.

*   **Что запоминается**: Последний установленный уровень громкости и URL последней активной радиостанции.
*   **Запись**: Данные обновляются мгновенно при командах `SET_VOLUME`, `FM` или `FM_STOP`.
*   **Восстановление**: При старте Klipper (или после `FIRMWARE_RESTART`) модуль читает JSON. Если перед выключением радио играло — оно включится само.
*   **FM_STOP**: Если вы выключили радио вручную командой `FM_STOP`, оно **не включится** само при перезагрузке, так как URL в памяти будет стерт.

---

## 4. РЕЖИМ ТИШИНЫ (QUIET HOUR)

Модуль имеет встроенный планировщик, работающий по системным часам.

*   **Настройка**: Параметры `work_start` и `work_end` определяют окно, когда звук разрешен.
*   **Наступление ночи**: Как только время переходит границу `work_end`, модуль:
    1. Мгновенно убивает радио и голос.
    2. Полностью очищает очередь уведомлений (чтобы они не "всплыли" утром).
    3. Сбрасывает системную громкость в 0.
*   **Наступление утра**: В момент `work_start` модуль проверяет, было ли включено радио перед уходом в тишину. Если да — музыка начинает играть автоматически.
*   **Игнорирование команд**: Любые команды `VOICE`, пришедшие ночью, будут проигнорированы. Команда `FM` ночью лишь сохранит станцию в память, но не запустит звук.
*   Формат времени должен состоять из двух цифр часа и двух цифр минут, например 06:05

---

## 5. ПРИМЕРЫ СЦЕНАРИЕВ

| Ситуация | Поведение системы |
| :--- | :--- |
| **Играет FM, вызван VOICE** | FM тухнет -> Голос говорит на 100% -> FM возвращается сам. |
| **Вызван каскад из 10 VOICE** | Все 10 проигрываются по очереди -> Включается FM. |
| **Пропал интернет во время FM** | Уведомление о потере сети -> Процесс убивается до появления связи. |
| **Интернет восстановился** | Система обнаруживает сеть -> Автозапуск последней станции. |
| **Старт радио (Метаданные)** | Поэтапный вывод на экран (Станция -> Жанр -> Трек) с задержками. |
| **Смена трека (Шпион)** | `ffprobe` находит новое название -> Уведомление на ClipperScreen. |
| **VOICE говорит, вызван FM_STOP** | URL стирается из памяти. После конца речи FM не включится. |
| **VOICE говорит, вызван CLEAR_VOICE** | Речь обрывается. Если FM был активен — он вернется через 1 сек. |
| **Наступило время Work_End** | Радио обрывается, очередь VOICE чистится. Полный Mute (0%). |
| **Включение принтера днем** | Авто-восстановление громкости и станции из JSON-памяти. |
| **Включение принтера ночью** | Загрузка в режиме тишины (0%), автозапуск FM заблокирован до утра. |

## 6. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ
*   Плееры: `mpg123`, `ffmpeg` (ffplay, ffprobe).
*   Доступ: Пользователь Klipper должен состоять в группе `audio`.
*   Конфиг: Обязательно наличие секции `[voice]` в `printer.cfg`.

*   `~/voice_state.json` — Файл памяти (создается автоматически).
*   `/tmp/klipper_fm.log` — Временный лог потока радио.
